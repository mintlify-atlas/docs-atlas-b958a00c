The quickest way to run the Hands-On ML notebooks is directly in the cloud — no installation, no package management, just click a link and start coding. If you prefer to work locally, you can have a full environment running in about ten minutes using Anaconda. This page covers both paths.

## Cloud options (no installation required)

You can open any notebook in several hosted environments. Google Colab is the recommended option because it offers free GPU access, deep integration with Google Drive, and the most consistent experience with the book's code.

<Tabs>
  <Tab title="Google Colab">
    Google Colab is the recommended platform. It runs on Google's infrastructure, provides optional free GPU/TPU access, and requires only a Google account.

    [Open in Google Colab](https://colab.research.google.com/github/ageron/handson-ml3/blob/main/)

    <Warning>
      Colab provides a temporary environment. Any files you create or data you download will be deleted when the session ends. Download anything you want to keep, or save notebooks to your Google Drive.
    </Warning>
  </Tab>
  <Tab title="Kaggle">
    Kaggle Notebooks offer free GPU access and a persistent workspace tied to your Kaggle account. The handson-ml3 notebooks are available directly on Kaggle.

    [Open in Kaggle](https://homl.info/kaggle3/)

    <Note>
      Kaggle has not been fully tested with every notebook. Most code runs correctly, but you may encounter minor environment differences.
    </Note>
  </Tab>
  <Tab title="Binder">
    Binder builds a live, temporary Jupyter environment directly from the GitHub repository. No account required.

    [Launch in Binder](https://mybinder.org/v2/gh/ageron/handson-ml3/HEAD?filepath=%2Findex.ipynb)

    <Note>
      Binder sessions are temporary and can be slow to start. It has not been fully tested with every notebook.
    </Note>
  </Tab>
  <Tab title="Deepnote">
    Deepnote provides a collaborative notebook environment with persistent storage and team features.

    [Launch in Deepnote](https://homl.info/deepnote3/)

    <Note>
      Deepnote has not been fully tested with every notebook in this repository.
    </Note>
  </Tab>
</Tabs>

## Local setup (3 steps)

Running locally gives you full control over your environment, persistent storage, and the ability to work offline. The setup uses Anaconda (or Miniconda) to create an isolated Python 3.10 environment with all required libraries.

<Steps>
  <Step title="Clone the repository and install Anaconda">
    Install [Anaconda](https://www.anaconda.com/products/distribution) or [Miniconda](https://docs.conda.io/en/latest/miniconda.html) if you do not have it already. Then clone the repository:

    ```bash
    git clone https://github.com/ageron/handson-ml3.git
    cd handson-ml3
    ```

    You will also need [git](https://git-scm.com/downloads). Alternatively, download [main.zip](https://github.com/ageron/handson-ml3/archive/main.zip) and unzip it manually.
  </Step>
  <Step title="Create and activate the conda environment">
    From inside the `handson-ml3` directory, create the `homl3` environment from the included `environment.yml` file. This installs Python 3.10 along with Scikit-Learn, TensorFlow, Keras, and all other required libraries:

    ```bash
    conda env create -f environment.yml
    conda activate homl3
    ```
  </Step>
  <Step title="Register the kernel and launch Jupyter">
    Register the `homl3` environment as a Jupyter kernel so the notebooks use the correct Python installation:

    ```bash
    python -m ipykernel install --user --name=python3
    jupyter notebook
    ```

    Your browser will open Jupyter's file browser. Click `index.ipynb` to see all chapters and start navigating.
  </Step>
</Steps>

<Tip>
  If you have an NVIDIA GPU with Compute Capability 3.5 or higher, you can enable GPU acceleration for TensorFlow. See the [full installation guide](/setup/installation) for GPU setup steps including CUDA and cuDNN.
</Tip>

## Prerequisites checklist

Before you start, make sure you have the following:

- **Python** — the conda environment installs Python 3.10 automatically
- **git** — needed to clone the repository (or download the zip instead)
- **Anaconda or Miniconda** — for local setup only
- **A Google account** — for Colab only
- **Basic Python knowledge** — familiarity with functions, loops, and importing libraries

No prior machine learning experience is required. The book and notebooks build up every concept from scratch.
